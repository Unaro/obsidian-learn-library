## 🌐 Сети и HTTP

### Из файла [[05_Infrastructure/01_Infrastructure_Networks_Docker]]

#### 1. HTTP
- **Stateless** — каждый запрос независим
- **Идемпотентные методы:**
  - `GET`, `PUT`, `DELETE`, `HEAD`
- **Не идемпотентные:**
  - `POST`, `PATCH`

#### 2. REST
- Ресурсы через URI
- Стандартные HTTP-методы
- JSON/XML обмен

#### 3. CORS
- Механизм безопасности браузера
- Заголовок: `Access-Control-Allow-Origin`

#### 4. JWT (JSON Web Token)
- **Структура:** `Header.Payload.Signature`
- **Header:** тип и алгоритм
- **Payload:** ID пользователя, срок действия
- **Signature:** проверка подлинности

## 📊 Системный анализ и нотации

### Из файла [[05_Infrastructure/02_System_Analysis_BPMN_UML]]

#### 1. BPMN 2.0 (Business Process Model and Notation)
- **Назначение:** моделирование бизнес-процессов во времени
- **Элементы:**
  - **События** (круги) — стартовые, промежуточные, конечные
  - **Задачи** (прямоугольники) — шаги процесса
  - **Шлюзы** (ромбы) — ветвление (XOR, AND, OR)
  - **Пулы и дорожки** — зоны ответственности

#### 2. IDEF0 (Функциональное моделирование)
- **Назначение:** что делает система (без привязки ко времени)
- **Принцип ICOM:**
  - **I**nput (Вход ← слева) — данные для обработки
  - **C**ontrol (Управление ↑ сверху) — правила и ограничения
  - **O**utput (Выход → справа) — результат
  - **M**echanism (Механизм ↓ снизу) — ресурсы

#### 3. UML (Unified Modeling Language)
| Диаграмма | Описание |
|-----------|----------|
| **Use Case** | Взаимодействие акторов с системой |
| **Class Diagram** | Классы, атрибуты, методы, связи |
| **Sequence Diagram** | Временная последовательность сообщений |

## 💻 TypeScript

### Из файла [[05_Infrastructure/03_TypeScript_Security_Testing]]

#### 1. interface vs type
- **interface** — для объектов, поддерживает слияние и extends
- **type** — универсален (примитивы, union, intersection)

#### 2. Generics
- Переменные для типов `<T>`
- Переиспользуемые компоненты с типизацией

#### 3. Utility Types
- `Partial<T>` — все свойства опциональны
- `Omit<T, K>` — исключить свойство K
- `Pick<T, K>` — оставить свойство K
- `Record<K, T>` — словарь

#### 4. Type Guarding
- `typeof`, `in`
- Кастомные предикаты (`value is Type`)
