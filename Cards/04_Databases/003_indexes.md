# Зачем нужны индексы в БД?

## Вопрос
Зачем нужны индексы в базах данных? Какие есть плюсы и минусы?
?
## Ответ

**Индексы** — структуры данных для ускорения поиска.

**B-Tree (по умолчанию в PostgreSQL):**
- Сбалансированное дерево
- Поиск за **O(log n)**
- Для 1 млн записей: ~20 сравнений

**Плюсы:**
- ✅ Ускоряют `SELECT`, `WHERE`, `ORDER BY`
- ✅ Ускоряют `JOIN` по ключевым полям
- ✅ Позволяют быстро находить диапазоны

**Минусы:**
- ❌ Занимают место на диске
- ❌ Замедляют `INSERT`, `UPDATE`, `DELETE`
- ❌ Требуют перестройки при изменении данных

**Trade-off:**
```
Без индекса:
├─ SELECT: ~500мс (полное сканирование)
└─ INSERT: ~1мс

С индексом:
├─ SELECT: ~5мс (O(log n)) ✅
└─ INSERT: ~5мс (обновление дерева) ⚠️
```

**Когда не индексировать:**
- Маленькие таблицы (< 1000 строк)
- Колонки с низкой селективностью (пол, статус)
- Часто изменяемые колонки
