# Что такое GIL?

## Вопрос
Что такое GIL в Python? Как он влияет на многопоточность?
?
## Ответ

**GIL (Global Interpreter Lock)** — мьютекс в CPython, позволяющий только одному потоку выполнять байт-код Python в любой момент времени.

**Влияние на многопоточность:**
- ✅ **I/O-bound задачи** (сеть, файлы) — работают хорошо, поток отдаёт GIL при ожидании
- ❌ **CPU-bound задачи** (вычисления) — не ускоряются, только один поток работает

**Решения:**
1. **multiprocessing** — обходит GIL, использует несколько процессов
2. **C расширения** — могут отпускать GIL
3. **Другие интерпретаторы** — Jython, IronPython (без GIL)

**Пример:**
```python
# I/O-bound — threading работает
import threading
for url in urls:
    threading.Thread(target=fetch, args=(url,)).start()

# CPU-bound — multiprocessing нужен
from multiprocessing import Process
for data in datasets:
    Process(target=process, args=(data,)).start()
```
