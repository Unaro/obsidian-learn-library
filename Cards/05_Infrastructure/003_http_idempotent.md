# Идемпотентные HTTP методы

## Вопрос
Что такое идемпотентность HTTP методов? Какие методы идемпотентны?
?
## Ответ

**Идемпотентность** — свойство, при котором повторный запрос не меняет состояние сервера.

**Идемпотентные методы:**

| Метод | Идемпотентный | Описание |
|-------|--------------|----------|
| `GET` | ✅ Да | Получение данных |
| `PUT` | ✅ Да | Полная замена ресурса |
| `DELETE` | ✅ Да | Удаление ресурса |
| `HEAD` | ✅ Да | Получить заголовки |
| `OPTIONS` | ✅ Да | Узнать доступные методы |

**Не идемпотентные:**

| Метод | Идемпотентный | Описание |
|-------|--------------|----------|
| `POST` | ❌ Нет | Создание ресурса |
| `PATCH` | ❌ Нет | Частичное обновление |

**Пример:**
```
PUT /users/123
{ "name": "John" }

# Один вызов: пользователь обновлён
# Десять вызовов: пользователь обновлён (тот же результат)

POST /users
{ "name": "John" }

# Один вызов: создан 1 пользователь
# Десять вызовов: создано 10 пользователей!
```

**Зачем важно:**
- Безопасные повторные запросы
- Кэширование
- Обработка ошибок
