# Файл 14. React Под Капотом и Оптимизация

## 1. Virtual DOM vs Shadow DOM

- **Virtual DOM (React):** Легковесная JavaScript-репрезентация реального DOM. React создает новое V-DOM дерево при изменении состояния, сравнивает его со старым (Diffing) и точечно обновляет только измененные узлы в реальном браузере.
    
- **Shadow DOM (Браузерная технология):** Механизм инкапсуляции веб-компонентов (изоляция HTML и CSS). Никак не связан с React, но их часто путают из-за названия на собеседованиях.
    

## 2. React Fiber и Алгоритм Согласования (Reconciliation)

- **Reconciliation:** Процесс, с помощью которого React понимает, какие части DOM нужно обновить (тот самый Diffing-алгоритм). Он работает за `O(n)` времени благодаря предположению, что элементы с разными типами производят разные деревья, и использованию пропса `key`.
    
- **React Fiber:** Переписанный движок согласования (начиная с React 16). Его главная фишка — способность разбивать работу рендеринга на мелкие части (chunks) и приостанавливать/возобновлять их (асинхронный рендеринг). Это позволяет не блокировать главный поток (Event Loop) тяжелыми вычислениями.
    

## 3. Компоненты высшего порядка (HOC)

- **Концепция:** HOC (Higher-Order Component) — это функция, которая принимает компонент и возвращает новый, "улучшенный" компонент (паттерн Декоратор).
    
- **Применение:** Переиспользование логики. Например, `withAuth(Dashboard)` — функция обернет дашборд и не пустит пользователя, если он не авторизован.
    
- **Отличие от хуков:** Хуки переиспользуют логику работы с состоянием внутри функциональных компонентов. HOC оборачивает сами компоненты (использовалось повсеместно до появления хуков, но актуально и сейчас).
    

## 4. React Portals

- **Зачем нужны:** Позволяют рендерить дочерние элементы в DOM-узел, который находится вне иерархии родительского компонента (например, напрямую в `document.body`).
    
- **Кейсы:** Модальные окна, тултипы, выпадающие меню. Это решает проблему, когда родительский блок имеет CSS-свойства `overflow: hidden` или низкий `z-index`, которые обрезают всплывающее окно.
    
- **Синтаксис:** `ReactDOM.createPortal(child, container)`.
    

## 5. Производительность: Оптимизация и Загрузка

- **Lazy Loading и Suspense:** Разделение кода (Code Splitting). Компоненты (например, тяжелые страницы или графики) загружаются не сразу в основном JS-бандле, а отдельными файлами (чанками) только в момент перехода к ним.

```JavaScript
	const HeavyComponent = React.lazy(() => import('./HeavyComponent'));
	<Suspense fallback={<Spinner />}><HeavyComponent /></Suspense>
```

- **Shimmer UI (Скелетные экраны):** Паттерн UX, когда вместо крутящегося спиннера показывается серый "скелет" будущего контента, имитирующий структуру данных. Психологически воспринимается пользователями как более быстрая загрузка.
