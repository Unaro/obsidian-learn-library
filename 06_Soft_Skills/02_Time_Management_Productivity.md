# Тайм-менеджмент и Продуктивность

> **Назначение:** Техники управления временем, приоритизация задач, борьба с прокрастинацией и выгоранием. Включает шаблоны и инструменты для разработчиков.

---

## 1. Приоритизация задач

### 1.1. Матрица Эйзенхауэра

| | **Срочно** | **Не срочно** |
|---|---|---|
| **Важно** | 🔴 СДЕЛАТЬ СЕЙЧАС | 🟡 ЗАПЛАНИРОВАТЬ |
| **Не важно** | 🟠 ДЕЛЕГИРОВАТЬ | 🟢 УДАЛИТЬ |

**Примеры для разработчика:**

| 🔴 Сделать | 🟡 Запланировать | 🟠 Делегировать | 🟢 Удалить |
|---|---|---|---|
| Баг в проде | Рефакторинг | Рутинные задачи | Лишние встречи |
| Таймауты API | Обучение | Code review (частично) | "Nice to have" фичи |
| Hotfix | Документация | Встречи без вас | Постоянные уведомления |

---

### 1.2. RICE Score для приоритизации

```
RICE = (Reach × Impact × Confidence) / Effort

Reach — сколько пользователей затронет
Impact — влияние на метрики (0.25-3)
Confidence — уверенность в оценке (0-100%)
Effort — усилия в человеко-неделях

Пример:
Фича A: (1000 × 2 × 80%) / 5 = 320
Фича B: (500 × 3 × 90%) / 3 = 450 ← Приоритетнее
```

### 2.2. Pomodoro для программирования

**Классический Pomodoro:**
```
25 мин работа → 5 мин перерыв → повторить 4 раза → 30 мин перерыв
```

**Адаптированный для разработки:**
```
50 мин работа → 10 мин перерыв → повторить 3 раза → 30 мин перерыв

Почему дольше:
• Вход в flow занимает 15-20 минут
• 25 минут недостаточно для сложной задачи
• Контекстное переключение дорого стоит
```

**Инструменты:**
- **Forest** — геймификация (растёт дерево пока фокусируешься)
- **Focus To-Do** — Pomodoro + список задач
- **Toggl Track** — трекинг времени по проектам

## 3. Борьба с прокрастинацией

### 3.1. Техники

**Правило 2 минут:**
```
Если задача занимает < 2 минут — сделай немедленно
• Ответить на email
• Закоммитить изменения
• Ответить в Slack
```

**Правило 5 минут:**
```
"Я поработаю над этим только 5 минут"

Психологический трюк:
• Начать легко (всего 5 минут)
• Часто продолжаешь дольше
• Инерция движения включается
```

**Eat The Frog:**
```
Самую неприятную задачу — первой с утра

Почему работает:
• Willpower исчерпаем к концу дня
• Избегание создаёт ментальную нагрузку
• После "лягушки" всё остальное легко
```

## 4. Борьба с выгоранием

### 4.1. Симптомы

**Ранние признаки:**
```
🟡 Эмоциональные:
• Раздражительность на код-ревью
• Цинизм по отношению к проекту
• "Меня это больше не волнует"

🟡 Поведенческие:
• Прокрастинация на задачах
• Избегание встреч
• Работа в нерабочее время

🟡 Когнитивные:
• Трудности с концентрацией
• Забывчивость на стендапах
• Ошибки в простом коде
```

**Поздние признаки:**
```
🔴 Физические:
• Хроническая усталость
• Проблемы со сном
• Частые болезни

🔴 Эмоциональные:
• Чувство бесполезности
• Потеря мотивации
• Депрессивные состояния
```

---

### 4.2. Профилактика

**На индивидуальном уровне:**
```
1. Границы:
   • Чёткое начало и конец дня
   • No-meeting дни (например, среда)
   • Отключение уведомлений после 18:00

2. Восстановление:
   • Минимум 1 полный выходной без кода
   • Отпуск каждые 3-4 месяца
   • Хобби вне IT (спорт, музыка, искусство)

3. Здоровье:
   • 7-8 часов сна
   • Регулярная физическая активность
   • Правильное питание
```

**На уровне команды:**
```
• Психологическая безопасность
• Регулярные 1-на-1 с менеджером
• Прозрачность ожиданий
• Поддержка work-life balance
• Признание достижений
```
