> **Назначение:** Краткие ответы на часто задаваемые вопросы и ключевые концепции

## 🔥 Топ-50 вопросов для подготовки

### Python

<details>
	<summary>
		<b>1. В чём разница между list и tuple?</b>
	</summary>
	- **list** — изменяемый, медленнее, `[]`
	- **tuple** — неизменяемый, быстрее, `()`
</details>

<details>
	<summary><b>2. Что такое GIL?</b></summary>

	**Global Interpreter Lock** — мьютекс в CPython, позволяющий только одному потоку выполнять байт-код Python одновременно.
</details>

<details>
<summary><b>3. Чем `is` отличается от `==`?</b></summary>

- `is` — проверяет идентичность объектов (один ли адрес в памяти)
- `==` — проверяет равенство значений
</details>

<details>
<summary><b>4. Что такое декоратор?</b></summary>

Функция, которая принимает другую функцию и расширяет её поведение без изменения её кода.
</details>

<details>
<summary><b>5. `*args` и `**kwargs` — что это?</b></summary>

- `*args` — переменное число позиционных аргументов (кортеж)
- `**kwargs` — переменное число именованных аргументов (словарь)
</details>

### Django

<details>
<summary><b>6. Что такое MVT?</b></summary>

**Model-View-Template** — вариация MVC в Django:
- Model — данные
- View — логика
- Template — отображение
</details>

<details>
<summary><b>7. Как оптимизировать N+1 запрос?</b></summary>

- `select_related()` — для ForeignKey (JOIN)
- `prefetch_related()` — для ManyToMany (раздельные запросы)
</details>

<details>
<summary><b>8. Что такое Middleware?</b></summary>

Промежуточное ПО, перехватывающее запросы до View и ответы после View.
</details>

<details>
<summary><b>9. В чём разница между FBV и CBV?</b></summary>

- **FBV** — Function-Based Views (простые функции)
- **CBV** — Class-Based Views (классы с методами get, post)
</details>

<details>
<summary><b>10. Что такое Serializer в DRF?</b></summary>

Класс для преобразования моделей Django в JSON и обратно.
</details>

### JavaScript

<details>
<summary><b>11. Что такое замыкание?</b></summary>

Способность функции запоминать и получать доступ к переменным из внешней области видимости, даже когда функция выполняется вне этой области.
</details>

<details>
<summary><b>12. Event Loop — как работает?</b></summary>

1. Выполняет синхронный код в **Call Stack**
2. Когда стек пуст, берёт задачу из **Microtask Queue** (Promise)
3. Затем из **Macrotask Queue** (setTimeout)
</details>

<details>
<summary><b>13. `var` vs `let` vs `const`?</b></summary>

- `var` — функциональная область видимости, всплывает
- `let` — блочная область видимости, TDZ
- `const` — как `let`, но нельзя переназначить
</details>

<details>
<summary><b>14. Что такое hoisting?</b></summary>

Механизм, при котором объявления переменных и функций "поднимаются" в верх области видимости перед выполнением кода.
</details>

<details>
<summary><b>15. Стрелочные функции vs обычные?</b></summary>

- Стрелочные — нет своего `this`, лексический контекст
- Обычные — `this` определяется контекстом вызова
</details>

### Node.js

<details>
<summary><b>16. Какие фазы Event Loop в Node.js?</b></summary>

1. Timers
2. Pending Callbacks
3. Idle, Prepare
4. **Poll** (I/O)
5. Check (`setImmediate`)
6. Close Callbacks
</details>

<details>
<summary><b>17. Что такое Backpressure?</b></summary>

Механизм регулировки скорости в Streams, когда быстрый источник приостанавливается, пока медленный приёмник не освободится.
</details>

<details>
<summary><b>18. Типы Streams?</b></summary>

- **Readable** — чтение
- **Writable** — запись
- **Duplex** — чтение и запись
- **Transform** — модификация данных
</details>

<details>
<summary><b>19. Для чего нужны Worker Threads?</b></summary>

Для **CPU-bound** задач, чтобы не блокировать главный Event Loop.
</details>

<details>
<summary><b>20. `process.nextTick()` vs `setImmediate()`?</b></summary>

- `process.nextTick()` — выполняется до следующей фазы Event Loop (абсолютный приоритет)
- `setImmediate()` — выполняется в фазе Check
</details>

---

### React

<details>
<summary><b>21. Virtual DOM vs Shadow DOM?</b></summary>

- **Virtual DOM** — JS-копия DOM для оптимизации (React)
- **Shadow DOM** — инкапсуляция веб-компонентов (браузерная технология)
</details>

<details>
<summary><b>22. Что такое React Fiber?</b></summary>

Движок согласования, разбивающий рендеринг на части (chunks) для асинхронного выполнения без блокировки Event Loop.
</details>

<details>
<summary><b>23. Controlled vs Uncontrolled компоненты?</b></summary>

- **Controlled** — значение в state React
- **Uncontrolled** — значение в DOM (через `useRef`)
</details>

<details>
<summary><b>24. Что такое Server Components?</b></summary>

Компоненты, выполняемые только на сервере, не попадают в клиентский бандл.
</details>

<details>
<summary><b>25. useMemo vs useCallback?</b></summary>

- `useMemo` — кэширует **значение**
- `useCallback` — кэширует **функцию**
</details>

---

### Next.js

<details>
<summary><b>26. SSR vs SSG vs CSR?</b></summary>

- **SSR** — генерация при каждом запросе
- **SSG** — генерация на этапе сборки
- **CSR** — рендеринг в браузере
</details>

<details>
<summary><b>27. 4 уровня кэширования в Next.js?</b></summary>

1. Request Memoization
2. Data Cache
3. Full Route Cache
4. Router Cache
</details>

<details>
<summary><b>28. Что такое Server Actions?</b></summary>

Асинхронные функции на сервере, вызываемые из клиентских компонентов (для форм без API routes).
</details>

<details>
<summary><b>29. revalidatePath vs revalidateTag?</b></summary>

- `revalidatePath('/blog')` — сброс кэша по URL
- `revalidateTag('posts')` — сброс кэша по тегу
</details>

<details>
<summary><b>30. Что такое `'use client'`?</b></summary>

Директива для Client Components (с интерактивностью: useState, useEffect, события).
</details>

---

### TypeScript

<details>
<summary><b>31. interface vs type?</b></summary>

- **interface** — для объектов, поддерживает слияние и extends
- **type** — универсален (примитивы, union, intersection)
</details>

<details><summary><b>32. Что такое Generics?</b></summary>

Переменные для типов `<T>`, позволяющие создавать переиспользуемые компоненты с сохранением типизации.
</details>

<details>
<summary><b>33. Utility Types?</b></summary>

- `Partial<T>` — все свойства опциональны
- `Omit<T, K>` — исключить свойство K
- `Pick<T, K>` — оставить свойство K
- `Record<K, T>` — словарь
</details>

<details>
<summary><b>34. Type Guard — что это?</b></summary>

Способ сузить тип: `typeof`, `in`, кастомные предикаты (`value is Type`).
</details>

### Базы Данных

<details>
<summary><b>35. Что такое ACID?</b></summary>

- **A**tomicity — неделимость
- **C**onsistency — согласованность
- **I**solation — изолированность
- **D**urability — устойчивость
</details>

<details>
<summary><b>36. Уровни изоляции транзакций?</b></summary>

1. READ UNCOMMITTED
2. READ COMMITTED
3. REPEATABLE READ
4. SERIALIZABLE
</details>

<details>
<summary><b>37. Зачем нужны индексы?</b></summary>

Ускоряют SELECT и WHERE (O(log n)), но замедляют INSERT/UPDATE/DELETE.
</details>

<details>
<summary><b>38. Проблема N+1 и как решить?</b></summary>

N+1 запрос при использовании ORM. Решение: JOIN или DataLoader.
</details>

<details>
<summary><b>39. Что такое JSONB?</b></summary>

Binary JSON в PostgreSQL — бинарный формат для хранения полуструктурированных данных.
</details>

<details>
<summary><b>40. CTE — что это?</b></summary>

Common Table Expressions (`WITH`) — временные именованные наборы результатов для улучшения читаемости запросов.
</details>

### Инфраструктура и сети

<details>
<summary><b>41. Образ vs Контейнер?</b></summary>

- **Образ** — статический шаблон
- **Контейнер** — запущенный экземпляр
</details>

<details>
<summary><b>42. git merge vs git rebase?</b></summary>

- **merge** — создаёт коммит слияния, сохраняет историю
- **rebase** — переписывает историю, линейная
</details>

<details>
<summary><b>43. Идемпотентные HTTP методы?</b></summary>

GET, PUT, DELETE, HEAD (повторный запрос не меняет состояние).
</details>

<details>
<summary><b>44. Что такое JWT?</b></summary>

JSON Web Token: Header.Payload.Signature — для Stateless аутентификации.
</details>

<details>
<summary><b>45. CORS — зачем?</b></summary>

Механизм безопасности браузера, блокирующий запросы к другим доменам без разрешения.
</details>

### Паттерны и архитектура

<details>
<summary><b>46. SOLID — расшифровка?</b></summary>

- **S**RP — Single Responsibility
- **O**CP — Open/Closed
- **L**SP — Liskov Substitution
- **I**SP — Interface Segregation
- **D**IP — Dependency Inversion
</details>

<details>
<summary><b>47. Dependency Injection?</b></summary>

Зависимости передаются извне (через конструктор), а не создаются внутри класса.
</details>

<details>
<summary><b>48. Singleton?</b></summary>

Гарантирует один экземпляр класса и глобальную точку доступа.
</details>

<details>
<summary><b>49. Пирамида тестирования?</b></summary>

1. **Unit** (основание) — много, быстрые
2. **Integration** — меньше
3. **E2E** (вершина) — мало, медленные
</details>

<details>
<summary><b>50. Agile vs Scrum?</b></summary>

- **Agile** — философия гибкой разработки
- **Scrum** — конкретный фреймворк (спринты, ритуалы)
</details>

## 💡 Метод STAR для поведенческих вопросов

Когда просят рассказать о сложной задаче:

| Буква | Значение | Пример |
|-------|----------|--------|
| **S** | Situation (Ситуация) | "Загружали Excel до 100K строк..." |
| **T** | Task (Задача) | "Нужно разгрузить сервер..." |
| **A** | Action (Действия) | "Внедрил IndexedDB, AST-парсер, Zustand..." |
| **R** | Result (Результат) | "Браузер не зависает, нагрузка снизилась на X%" |
