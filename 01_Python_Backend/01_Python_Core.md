# Файл 1. Теория Python (Core)

## 1. Типы данных и структуры

- **Мутабельность:**
    - _Неизменяемые (Immutable):_ `int`, `float`, `complex`, `bool`, `tuple`, `str`, `frozenset`. При изменении создается новый объект в памяти.
    - _Изменяемые (Mutable):_ `list`, `set`, `dict`. Изменяются на месте без изменения идентификатора объекта.

- **Списки (`list`) vs Кортежи (`tuple`):** Списки объявляются квадратными скобками и являются изменяемыми. Кортежи объявляются круглыми скобками, неизменяемы и за счет этого работают быстрее списков.
- **Словари (`dict`):** Неупорядоченные коллекции пар ключ-значение. Обеспечивают быстрый доступ за константное время O(1) благодаря использованию хэш-таблиц. Ключами могут быть только неизменяемые типы.
    
- **Копирование:**
    - `copy()`: Поверхностная копия. Копирует только ссылки на вложенные элементы (изменение вложенного списка в копии изменит оригинал).
    - `deepcopy()`: Глубокая копия. Рекурсивно копирует все элементы, создавая полностью независимый объект.

## 2. Функции, Итераторы и Генераторы

- **Аргументы функций:**
    - `*args`: Позволяет передавать переменное количество позиционных аргументов (передаются в виде кортежа).
    - `**kwargs`: Позволяет передавать переменное количество именованных аргументов (передаются в виде словаря).

- **Lambda-функции:** Анонимные функции, определяемые в одну строку без ключевого слова `def`, содержащие только одно выражение.
- **Итераторы vs Генераторы:**
    - _Итератор:_ Любой объект, реализующий методы `__iter__` и `__next__`. Работает только с итерируемым объектом и экономно потребляет память.
    - _Генератор:_ Функция с оператором `yield`, возвращающая объект-генератор. Сохраняет состояние локальных переменных при каждой приостановке.

- **Область видимости (LEGB):** Порядок поиска имен переменных интерпретатором:
    1. **L**ocal (Локальная): Внутри текущей функции.
    2. **E**nclosing (Замыкание): В области видимости внешней (оборачивающей) функции.
    3. **G**lobal (Глобальная): На уровне модуля/файла.
    4. **B**uilt-in (Встроенная): Встроенные имена Python (`print`, `len` и т.д.).

## 3. Объектно-ориентированное программирование (ООП)

- **`__new__` vs `__init__`:**
    - `__new__`: Статический метод, вызывается первым. Отвечает за выделение памяти и создание нового экземпляра. Возвращает объект.
    - `__init__`: Метод экземпляра, вызывается после `__new__`. Инициализирует атрибуты созданного объекта, ничего не возвращает.

- **Инкапсуляция (Подчеркивания):**
    - Одинарное (`_`): Конвенция, указывающая, что атрибут предназначен для внутреннего использования (protected).
    - Двойное (`__`): Включает name mangling (манглинг имен), добавляя префикс `_ИмяКласса`, защищая атрибут от переопределения в подклассах (private).

- **Декораторы методов класса:**
    - `@classmethod`: Метод принимает ссылку на класс (`cls`) первым аргументом. Работает с классом в целом.
    - `@staticmethod`: Статический метод. Не принимает ни `self`, ни `cls`. Вызывается через класс без доступа к его состоянию.
    - `@property`: Превращает метод в атрибут, позволяя реализовать геттеры, сеттеры (`@name.setter`) и делитеры для контроля доступа к данным.

- **Дандер-методы (Магические методы):** Методы, обрамленные двойными подчеркиваниями (например, `__str__`, `__add__`), автоматически вызываемые интерпретатором для реализации специального поведения (арифметика, строковое представление).
- **Абстрактные классы:** Служат базовым шаблоном, от которого нельзя создать экземпляр. Реализуются через модуль `abc` и декоратор `@abstractmethod`. Подклассы обязаны предоставить реализацию абстрактных методов.


## 4. Память и Операторы

- **`is` vs ` ==`:**
    - `is`: Проверяет идентичность объектов (ссылаются ли переменные на один и тот же участок памяти). Возвращает `True`, если `id(a) == id(b)`.
    - ` ==`: Проверяет равенство значений объектов, независимо от их адреса в памяти.

- **`id()`:** Возвращает уникальный целочисленный идентификатор объекта, гарантированно уникальный во время его существования.
- **Передача аргументов:** Аргументы передаются по ссылке на объект. Для изменяемых типов функция может изменить оригинал; для неизменяемых — изменения создадут локальную копию.

## 5. Конкурентность и Многопоточность

- **GIL (Global Interpreter Lock):** Мьютекс в CPython, позволяющий только одному потоку выполнять байт-код Python в любой момент времени. Защищает от состояний гонки при управлении памятью, но делает потоки неэффективными для вычислительно сложных (CPU-bound) задач.
- **Потоки (`threading`):** Легковесные процессы, использующие общую память в пределах одного процесса. Эффективны для I/O-bound задач (сетевые запросы, чтение файлов), когда поток блокируется и отдает GIL.
- **Процессы (`multiprocessing`):** Независимые сущности с собственным пространством памяти и своим интерпретатором Python. Позволяют обходить GIL и выполнять параллельные вычисления на разных ядрах процессора.
