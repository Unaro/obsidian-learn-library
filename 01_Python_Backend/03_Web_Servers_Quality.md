# Файл 6. Web-серверы, Real-time и Качество кода

## 1. Кэширование

- **Концепция:** Сохранение временных копий данных или результатов вычислений для ускорения доступа и снижения нагрузки на систему.
- **Преимущества:** Улучшение производительности, сокращение обращений к БД, экономия вычислительных ресурсов.
- **Недостатки:** Риск предоставления устаревших данных, затраты памяти, сложность инвалидации (поддержания согласованности) кэша.
- **Стратегии кэширования в Django:**
    - _Memcached / Redis:_ Кэширование в оперативной памяти. Самый быстрый и эффективный способ.        
    - _Local-memory Caching:_ Стратегия по умолчанию в Django, использует локальную память сервера.
    - _FileSystem Caching:_ Сериализованные данные хранятся в виде отдельных файлов на диске.
    - _Database Caching:_ Хранение кэша в таблице базы данных (эффективно при быстром сервере БД).

## 2. Взаимодействие веб-сервера и приложения (WSGI / ASGI)

- **WSGI (Web Server Gateway Interface):** Стандарт интерфейса для синхронного взаимодействия между веб-сервером и веб-приложением на Python. Позволяет серверу передавать HTTP-запросы приложению и получать ответы.
- **Популярные WSGI-серверы:**
    - _Gunicorn:_ Простой, стабильный и производительный сервер для обработки WSGI-приложений.
    - _uWSGI:_ Высокопроизводительный сервер с широким набором функций и поддержкой различных протоколов.

- **ASGI (Asynchronous Server Gateway Interface):** Современный стандарт, поддерживающий асинхронную обработку запросов. Точка входа находится в файле `asgi.py`.

## 3. WebSockets и Django Channels

- **Ограничения стандарта:** Стандартный Django работает синхронно, обрабатывая запросы последовательно, что не подходит для real-time задач.
- **Django Channels:** Официальное расширение, добавляющее поддержку WebSockets и асинхронных протоколов.
- **Возможности:**
    - _WebSocket поддержка:_ Двунаправленное соединение в реальном времени между клиентом и сервером (идеально для чатов, уведомлений, игр).
    - _Каналы (Channels):_ Механизм обмена сообщениями.
    - _Асинхронные задачи и представления (Views):_ Позволяет обрабатывать множество запросов одновременно и выполнять долгие фоновые операции без блокировки основного потока.

## 4. Качество кода и Методологии разработки

- **PEP 8:** Официальное руководство по стилю написания кода на Python. Регламентирует отступы, именование переменных, длину строк, что обеспечивает единый стиль, читаемость и удобство совместной работы.
- **Критерии хорошего кода:** Читаемость, простота (без излишних усложнений), эффективность, масштабируемость, надежность (обработка ошибок), тестируемость и безопасность.
    
- **TDD (Test-Driven Development):** Разработка через тестирование. Процесс состоит из трех жестких этапов:
    1. _Red (Написание теста):_ Пишется тест для несуществующего функционала (тест закономерно падает).
    2. _Green (Написание кода):_ Пишется минимальный код, чтобы тест успешно прошел.
    3. _Refactor (Рефакторинг):_ Код оптимизируется и улучшается без изменения его внешнего поведения.
	- _Итог:_ Уменьшение количества ошибок и создание самодокументируемого кода.
