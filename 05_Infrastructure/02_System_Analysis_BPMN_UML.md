## 1. BPMN 2.0 (Business Process Model and Notation)

### 1.1. Элементы BPMN

```
┌─────────────────────────────────────────────────────────┐
│              BPMN 2.0 Elements                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  СОБЫТИЯ (Events) — круги:                              │
│  ┌─────┐  Start Event     (начало процесса)            │
│  │     │                                                 │
│  └─────┘                                                 │
│  ┌─────┐  Intermediate Event (между действиями)        │
│  │ ~ ~ │                                                 │
│  └─────┘                                                 │
│  ┌─────┐  End Event       (конец процесса)             │
│  │ ●●● │                                                 │
│  └─────┘                                                 │
│                                                         │
│  ДЕЙСТВИЯ (Activities) — прямоугольники:                │
│  ┌─────────────┐                                        │
│  │   Задача    │  Task (атомарное действие)            │
│  └─────────────┘                                        │
│  ┌─────────────┐                                        │
│  │ Подпроцесс  │  Sub-process (с детализацией)         │
│  └─────────────┘                                        │
│                                                         │
│  ШЛЮЗЫ (Gateways) — ромбы:                              │
│  ┌─────┐  Exclusive Gateway (XOR — один путь)         │
│  │  X  │                                                 │
│  └─────┘                                                 │
│  ┌─────┐  Parallel Gateway (AND — все пути)           │
│  │  +  │                                                 │
│  └─────┘                                                 │
│  ┌─────┐  Inclusive Gateway (OR — один или больше)    │
│  │  O  │                                                 │
│  └─────┘                                                 │
│                                                         │
│  ПОТОКИ (Flows):                                        │
│  ─────>  Sequence Flow (порядок выполнения)            │
│  ─ ─ ─ > Message Flow (сообщения между пулами)         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.3. Типы шлюзов

**Exclusive Gateway (XOR):**
```
                    ┌─────┐
               ────>│  X  │────> Путь A (только если условие A)
                    └─────┤
                          └────> Путь B (только если условие B)

Пример:
- Оплата прошла? → Да/Нет (только один путь)
- Товар есть в наличии? → Да/Нет
```

**Parallel Gateway (AND):**
```
                    ┌─────┐
               ────>│  +  │────> Параллельно: Отправить email
                    └─────┤
                          └────> Параллельно: Создать накладную

Пример:
- После оплаты:
  - Отправить подтверждение
  - Передать на склад
  - Обновить статистику
```

**Inclusive Gateway (OR):**
```
                    ┌─────┐
               ────>│  O  │────> Путь A (если условие A)
                    └─────┤
                          └────> Путь B (если условие B)

Пример:
- Способ доставки:
  - Курьер (если выбран)
  - Самовывоз (если выбран)
  - Почта (если выбран)
  - Можно несколько одновременно
```

## 2. IDEF0 (Функциональное моделирование)

### 2.1. Принцип ICOM

```
┌─────────────────────────────────────────────────────────┐
│              IDEF0 Block Diagram                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    Control (Управление)                 │
│                    ──────────────────                   │
│                           ↓                             │
│                           ↓                             │
│         ┌─────────────────────────────┐                │
│  Input →│                             │→ Output        │
│  (Вход) │      ФУНКЦИЯ / ПРОЦЕСС      │→ (Выход)       │
│         │                             │                │
│         └─────────────────────────────┘                │
│                           ↑                             │
│                           ↑                             │
│                    Mechanism (Механизм)                 │
│                    ──────────────────                   │
│                                                         │

Пример:
Вход: Заказ клиента

Выход: Подтверждённый заказ
```

### 4.2. Рефакторинг с AI

**Промпт:**
```
Оптимизируй эту функцию для работы с большими массивами:

function findDuplicates(arr1, arr2) {
    const duplicates = [];
    for (let i = 0; i < arr1.length; i++) {
        for (let j = 0; j < arr2.length; j++) {
            if (arr1[i] === arr2[j]) {
                duplicates.push(arr1[i]);
            }
        }
    }
    return duplicates;
}
```

**AI предлагает:**
```typescript
// O(n*m) → O(n+m)
function findDuplicates(arr1, arr2) {
    const set2 = new Set(arr2);
    return arr1.filter(item => set2.has(item));
}

// Или для очень больших массивов:
function findDuplicates(arr1, arr2) {
    const set1 = new Set(arr1);
    const set2 = new Set(arr2);
    return [...set1].filter(x => set2.has(x));
}
```

### 4.3. Генерация тестов

**Промпт:**
```
Напиши unit тесты для этой функции:

async function fetchUser(id: string): Promise<User | null> {
    const response = await fetch(`/api/users/${id}`);
    
    if (!response.ok) {
        if (response.status === 404) return null;
        throw new Error('Failed to fetch user');
    }
    
    return response.json();
}
```

**AI генерирует:**
```typescript
import { fetchUser } from './fetchUser';

global.fetch = jest.fn();

describe('fetchUser', () => {
    beforeEach(() => {
        jest.clearAllMocks();
    });
    
    it('should return user data on success', async () => {
        const mockUser = { id: '1', name: 'John' };
        (global.fetch as jest.Mock).mockResolvedValue({
            ok: true,
            json: async () => mockUser
        });
        
        const result = await fetchUser('1');
        
        expect(result).toEqual(mockUser);
        expect(global.fetch).toHaveBeenCalledWith('/api/users/1');
    });
    
    it('should return null on 404', async () => {
        (global.fetch as jest.Mock).mockResolvedValue({
            ok: false,
            status: 404
        });
        
        const result = await fetchUser('999');
        
        expect(result).toBeNull();
    });
    
    it('should throw error on other failures', async () => {
        (global.fetch as jest.Mock).mockResolvedValue({
            ok: false,
            status: 500
        });
        
        await expect(fetchUser('1')).rejects.toThrow('Failed to fetch user');
    });
});
```
