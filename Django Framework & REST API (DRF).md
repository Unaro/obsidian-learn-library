# Файл 5. Django Framework & REST API (DRF)

## 1. Архитектура Django: Паттерн MVT

- **MVT (Model-View-Template):** Вариация паттерна MVC, используемая в Django.
    - **Model (Модель):** Отвечает за структуру базы данных, обработку данных и бизнес-логику.
    - **View (Представление):** Принимает HTTP-запрос, извлекает нужные данные через Модель и возвращает HTTP-ответ (или передает данные в Шаблон). Не обязательно занимается отрисовкой.
    - **Template (Шаблон):** Отвечает за HTML-разметку и отображение данных пользователю (заменяет классический View из MVC).

## 2. Django ORM и Оптимизация запросов

- **Типы связей:**
    - `OneToOneField`: Связь "один к одному". Каждая запись в одной модели связана только с одной записью в другой.
    - `ForeignKey`: Связь "один ко многим".
    - `ManyToManyField`: Связь "многие ко многим". Под капотом реализуется через создание дополнительной (промежуточной) невидимой таблицы связи.

- **Оптимизация (Проблема N+1):**
    - `select_related()`: Используется для связей `ForeignKey` и `OneToOneField`. Выполняет операцию `JOIN` на уровне SQL, забирая все связанные данные за _один_ запрос к БД.
    - `prefetch_related()`: Используется для `ManyToManyField` и обратных связей. Делает раздельные запросы к БД, а затем связывает объекты на уровне памяти Python.

- **`Q` объекты:** Позволяют создавать сложные запросы с использованием логических операторов `OR` (`|`) и `AND` (`&`), например, для поиска по нескольким условиям одновременно.

## 3. Middleware и Безопасность

- **Middleware (Промежуточное ПО):** Слой, который перехватывает и обрабатывает HTTP-запросы _до_ того, как они попадут во View, и HTTP-ответы _после_ выполнения View. Используется для аутентификации, логирования, кэширования.
- **CSRF Middleware:** Защищает от атак подделки межсайтовых запросов (Cross-Site Request Forgery).
    - _Алгоритм:_ Генерирует уникальный токен. Токен встраивается в HTML-формы или заголовки. При POST-запросе middleware проверяет совпадение токена; если он неверен — возвращает ошибку 403 Forbidden.


## 4. Представления (Views)

- **FBV (Function-Based Views):** Представления на основе обычных функций Python. Принимают `request` и возвращают `HttpResponse`.
- **CBV (Class-Based Views):** Представления на основе классов. Позволяют использовать принципы ООП (наследование, миксины). Методы внутри класса (`get`, `post`) автоматически маршрутизируют соответствующие HTTP-запросы.

## 5. Django REST Framework (DRF)

- **Назначение:** Мощный инструментарий для создания RESTful API поверх Django.
- **Serializer (Сериализатор):** Класс для преобразования сложных типов данных (моделей Django, QuerySet) в форматы типа JSON и обратно (десериализация).
    - _Валидация:_ При получении данных от клиента вызывается метод `is_valid()`. DRF проверяет типы полей, ограничения модели и кастомные валидаторы. При ошибке возвращается 400 Bad Request.

- **Views в DRF:**
    - `APIView`: Базовый класс для REST-представлений.
    - `GenericAPIView` + Mixins: Предоставляют готовое поведение для CRUD-операций.
    - `ViewSet`: Объединяет логику всех CRUD-операций (list, create, retrieve, update, destroy) для конкретной модели в одном классе. Используется совместно с Router для автоматической генерации URL-маршрутов.

- **Аутентификация в DRF:**
    - `SessionAuthentication`: На основе cookies (стандартный механизм Django).
    - `TokenAuthentication`: Сервер выдает уникальный токен, который клиент передает в заголовке `Authorization`.
    - `JWT Authentication`: Использование JSON Web Tokens. Токен содержит зашифрованные данные пользователя и проверяется без необходимости обращения к базе данных (Stateless).

- **Пагинация:** Позволяет разбивать большие объемы данных на части. Доступны `PageNumberPagination` (по номерам страниц), `LimitOffsetPagination` (смещение и лимит) и `CursorPagination` (для огромных массивов без смещения).
