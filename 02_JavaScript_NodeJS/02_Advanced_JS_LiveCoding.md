# Файл 13. Продвинутый JavaScript и Live-Coding (Под капотом)

## 1. Live-Coding: Замыкания и Оптимизация вызовов

Эти функции часто просят написать с нуля на виртуальной доске.

- **Debounce (Устранение дребезга):** Откладывает вызов функции до тех пор, пока не пройдет заданное время с момента последнего вызова (идеально для строки поиска).
```javascript
    function debounce(fn, delay) {
      let timeoutId;
      return function(...args) {
        clearTimeout(timeoutId);
        timeoutId = setTimeout(() => fn.apply(this, args), delay);
      };
    }
```

- **Throttle (Троттлинг):** Гарантирует, что функция будет вызываться не чаще, чем один раз в заданный интервал времени (идеально для отслеживания скролла или ресайза окна).
    
```JavaScript
    function throttle(fn, delay) {
      let shouldWait = false;
      return function(...args) {
        if (shouldWait) return;
        fn.apply(this, args);
        shouldWait = true;
        setTimeout(() => shouldWait = false, delay);
      };
    }
```

- **Currying (Каррирование):** Трансформация функции с множеством аргументов в набор вложенных функций с одним аргументом (`f(a, b, c)` -> `f(a)(b)(c)`).

```JavaScript
	function curry(fn) {
	  return function curried(...args) {
		if (args.length >= fn.length) {
		  return fn.apply(this, args);
		} else {
		  return function(...nextArgs) {
			return curried.apply(this, args.concat(nextArgs));
		  }
		}
	  };
	}
```

## 2. Глубокое копирование объектов (Deep Copy)

- **Поверхностное копирование (Shallow):** `Object.assign({}, obj)` или `{...obj}`. Копирует только первый уровень, вложенные объекты передаются по ссылке.
    
- **Глубокое копирование (Deep):** * _Старый способ:_ `JSON.parse(JSON.stringify(obj))` (теряет методы, `Date`, `undefined`, циклические ссылки).
    - _Новый стандарт:_ `structuredClone(obj)` (поддерживается современными браузерами, работает с `Date`, `Set`, `Map`).
    - _Рекурсивный полифил (часто просят написать):_ Проверка `typeof val !== 'object' || val === null` -> возврат значения. Иначе -> рекурсивный обход ключей для массивов и объектов.

## 3. Продвинутая асинхронность: Методы Promise

- `Promise.all(iterable)`: Ждет выполнения всех промисов. Если хотя бы один упадет (reject), падает весь `Promise.all`.
- `Promise.allSettled(iterable)`: Ждет завершения всех промисов, независимо от того, выполнились они успешно (fulfilled) или с ошибкой (rejected). Возвращает массив объектов со статусами и результатами/ошибками.
- `Promise.race(iterable)`: Возвращает результат первого завершенного промиса (неважно, resolve или reject).
- `Promise.any(iterable)`: Возвращает первый успешно выполненный промис (resolve). Если все упали — возвращает ошибку `AggregateError`.

## 4. События (Events) в браузере

- **Три фазы события:** 1. Погружение (Capturing - от Window до элемента) -> 2. Целевая фаза (Target) -> 3. Всплытие (Bubbling - от элемента обратно к Window).
    
- `event.target` vs `event.currentTarget`:
    - `target`: Исходный элемент, на котором физически произошло событие (например, клик по `<span>` внутри `<button>`).
    - `currentTarget`: Элемент, на котором в данный момент срабатывает обработчик события (сама `<button>`).

- **Делегирование событий:** Вместо навешивания 1000 обработчиков на каждый `<li>` в списке, вешается один обработчик на родительский `<ul>`. Внутри проверяется `event.target.tagName`. Улучшает производительность.

## 5. Управление памятью и Защита объектов

- **Garbage Collector (Алгоритм Mark-and-Sweep):** Сборщик мусора начинает от корня (Window) и помечает все достижимые объекты. Все непомеченные (недостижимые) объекты удаляются из памяти.
- **`WeakMap` и `WeakSet`:** В отличие от обычных `Map`/`Set`, они не предотвращают удаление своих ключей (которые могут быть только объектами) сборщиком мусора, если на них больше нет ссылок в коде. Идеально для кэширования и хранения временных данных.
- **Ограничение объектов:**
    - `Object.preventExtensions(obj)`: Запрещает добавление новых свойств.
    - `Object.seal(obj)`: Запрещает добавление/удаление свойств, но разрешает изменение существующих.
    - `Object.freeze(obj)`: Полная заморозка. Запрещает добавление, удаление и изменение свойств (делает объект иммутабельным на первом уровне).

# Файл 15. Архитектура React, Роутинг и Легаси

## 1. Классовые компоненты и Жизненный цикл (Legacy)

Хотя сейчас стандартом являются функциональные компоненты с хуками, знание классов демонстрирует глубину.

- `constructor(props)`: Инициализация `this.state` и привязка контекста (`bind`).
    
- **Mounting (Монтирование):**
    
    - `componentDidMount()`: Компонент отрисован в DOM. Идеальное место для API-запросов и подписок. (Аналог в хуках: `useEffect(..., [])`).
        
- **Updating (Обновление):**
    
    - `shouldComponentUpdate(nextProps, nextState)`: Позволяет отменить рендер для оптимизации (Аналог: `React.memo`).
        
    - `componentDidUpdate(prevProps, prevState)`: Срабатывает после обновления. Можно делать сайд-эффекты на основе изменений пропсов. (Аналог: `useEffect(..., [deps])`).
        
- **Unmounting (Размонтирование):**
    
    - `componentWillUnmount()`: Перед удалением компонента. Отписка от сокетов, очистка таймеров (`clearInterval`). (Аналог: функция `return () => {}` внутри `useEffect`).
        

## 2. Паттерны проектирования UI

- **Smart vs Dumb Components (Контейнеры и Презентационные):**
    
    - _Smart (Контейнеры):_ Содержат логику, делают запросы к API, управляют состоянием (`useState`/Zustand), передают данные вниз.
        
    - _Dumb (Презентационные):_ Вообще не имеют сложного состояния. Принимают данные через пропсы (props) и просто красиво их отрисовывают.
        
- **Controlled vs Uncontrolled (Работа с формами):**
    
    - _Controlled (Контролируемые):_ Значение инпута жестко привязано к состоянию React (`value={state} onChange={setState}`). React является единственным источником истины.
        
    - _Uncontrolled (Неконтролируемые):_ Значение хранится в самом DOM. React получает к нему доступ только по требованию через ссылки (`useRef`).
        

## 3. Роутинг и Безопасность (React Router)

- **SPA (Single Page Application):** Приложение загружает один HTML-файл. Переход по страницам не перезагружает вкладку браузера, а просто меняет компоненты местами с помощью JavaScript и HTML5 History API.
    
- **Protected Routes (Защищенные маршруты):** Компонент-обертка, который проверяет наличие токена авторизации. Если токена нет — происходит редирект (например, через `<Navigate to="/login" />`).
    
- **RBAC (Role-Based Access Control):** Расширение защищенных маршрутов. Проверяется не только факт входа, но и роль (Admin, User, Manager) для предоставления доступа к определенным дашбордам или кнопкам.
    

## 4. Инфраструктура: Vite vs Create React App (CRA)

- **CRA (Устарел):** Использовал Webpack под капотом. При любом изменении файла пересобирал весь бандл приложения, что делало разработку медленной на крупных проектах.
- **Vite (Современный стандарт):** Не собирает бандл при разработке (dev-режиме). Он отдает браузеру исходные файлы как нативные ES-модули (ESM). Пересобирается только тот конкретный файл, который был изменен (HMR — Hot Module Replacement). Это делает запуск локального сервера почти мгновенным.
